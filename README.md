# POTTER-FSM

> 状态管理和有限状态机

## 背景需求:

因为在不同的业务中，对一些实体对象都出现了状态管理和状态切换的需求。
我们希望抽象出一个状态管理和业务流管理的机制，便于业务实现。

实现要求: 实现一个工具库

功能说明
- 整个机制必须是配置化的，支持任意业务模型的。
- 支持对任意符合特定标准的对象进行状态管理
  - 状态数据 - 规范化，但同样有足够的扩展性
  - 状态行为
    - 在不同状态、不同条件下，执行相应不同的业务代码。
    - 需要三类行为：进入状态、处于状态、退出状态。
  - 状态控制
    - 根据状态数据，通过可配置化的条件进行可控的状态切换。
    - 可以预设一些通用的控制逻辑
